emo

Expo-like Android development framework in Go. Write .em files, see them live on Android. No JavaScript, no Gradle rebuilds.

Go 1.22+ Kotlin / Jetpack Compose v0.2.2 MIT License
$ curl -fsSL https://raw.githubusercontent.com/crossberry-in/emo/main/install.sh | bash

Quick Start

# Install emo
curl -fsSL https://raw.githubusercontent.com/crossberry-in/emo/main/install.sh | bash

# Create your first app
emo create my-app
cd my-app

# Start the dev server (live reload)
emo start

# In another terminal, launch on Android
emo go

Key features

  • .em custom language — Svelte/Vue-style single-file components with JSX-like syntax
  • 27 native UI elements — Column, Row, Text, Button, WebView, Switch, Slider, Card, Picker, TabBar, and more
  • Live reload < 150ms — edit .em/.css, save, device updates instantly
  • CSS styling — standard CSS with dp/sp units, className attributes
  • File-based routing — app/index.em, app/(tabs)/_layout.em, app/[id].em
  • Navigation API — Navigate, Back, Replace, dynamic route params
  • Android sync — auto-generates native Kotlin @Composable + XML resources
  • Production mode — embedded runtime runs on-device with no server
  • Plugin system — camera, location, storage, haptics, linking
  • emo Go preview app — auto-detects dev servers, carousel, virtual preview
  • 22 installable components — emo install WebView, emo install Card, etc.
  • 5 project templates — default, blank, counter, todo, navigation

💡 Tip: Run emo templates to see available project templates, or emo components to browse installable components from the emo-templates repo.
